# API Documentation Glossary

_This glossary is a living document._

A comprehensive glossary of API documentation terms, concepts, and best practices
built with Docusaurus. This project originated from coursework in the
University of Washington's AI-Enhanced API Documentation certification program.

## About this project

This glossary serves as a reference guide for technical writers, developers, and
anyone working with API documentation. It covers fundamental concepts, tools,
processes, and best practices in the API documentation space.

### Topics covered

- **API Fundamentals**: REST APIs, security, OpenAPI Specification
- **Documentation Types**: Overview topics, reference documentation, tutorials
- **Tools & Technologies**: cURL, Postman, Git/GitHub, Markdown
- **Processes**: Agile, DDLC, project management, usability testing
- **Concepts**: Cognitive dimensions of API usability, audience analysis, domain knowledge
- **Writing Style**: Rhetorical approaches, content strategy
- **AI & APIs**: Artificial intelligence in API documentation

## Getting started

[quick reference draft here](./docs/quick-reference.md)

### Prerequisites

- `Node.js` version 20.0+
- `npm` or `yarn` package manager

### Installation

```bash
# Clone the repository
git clone https://github.com/yourusername/api-docs-glossary.git

# Navigate to the project directory
cd api-docs-glossary

# Install dependencies
npm install
```

### Local Development

```bash
# Start the development server
npm run start
```

This command starts a local development server and opens up a browser
window at `http://localhost:3000/`. Most changes are reflected live
without having to restart the server.

### Build

```bash
# Create a production build
npm run build
```

This command generates static content into the `build` directory
that can be served using any static contents hosting service.
